Prerequisites & Setup Make sure your environment is ready

Step 1

Activate the Virtual Environment

Open a terminal in the project root and activate the Python virtual environment:

PS> cd C:\Projects\Advanced_Plan_Parser
PS> .\.venv\Scripts\Activate.ps1
Step 2

Verify Dependencies

Ensure the package and its dependencies are installed in editable mode:

PS> pip install -e .
# For optional ML backends:
PS> pip install -e ".[layout]"   # LayoutLMv3 (transformers + torch)
PS> pip install -e ".[gnn]"      # Graph Neural Network (torch-geometric)
ℹ️
The base install includes scikit-learn and joblib, which is all you need for the primary GBM classifier. The layout and gnn extras are optional advanced features.
Step 3

Prepare Your PDF

Place your construction plan PDF in the input/ folder (or anywhere accessible). Supported formats: multi-page PDF with raster or vector content.

💡
Higher quality scans produce better OCR results. Aim for 300 DPI or above. The pipeline's default render DPI is 300.

Step 4

Launch the GUI

Double-click launch_gui.bat in the project root, or run:

PS> python scripts/gui/gui.py

⌨️

4

Annotate & Correct Detections This is where your ML training data is created

🎯
This is the most important phase. Every correction you make here becomes a labeled training example. The quality and quantity of your annotations directly determine model accuracy.
Step 12

Open the Annotation Tab (Tab 6)

Press Ctrl+6 or click the Annotation tab. You'll see the PDF page rendered with color-coded bounding boxes overlaid on each detected region.

Step 13

Understand the Color Coding

Each element type has a distinct color so you can quickly assess detections at a glance:

Element TypeColorTypical Content
notes_columnBlueFull notes column region
notes_blockGreenIndividual note paragraphs
headerRedSection headers, titles
abbreviationsOrangeAbbreviation tables/lists
legendPurpleSymbol legends
revisionCyanRevision history blocks
standard_detailBrownStandard construction details
title_blockBlue-GreyTitle block / cartouche area
misc_titlePinkMiscellaneous titles/labels
Step 14

Inspect a Detection

Click any bounding box on the canvas. The Inspector Panel on the right shows:

  • Detection ID — unique identifier
  • Element Type — dropdown to change the label
  • Confidence — model's confidence score (if a model exists)
  • Text Content — OCR-extracted text within the box
  • Model Suggestion — "Model suggests: X (Y%)" with an Apply button
  • Group Membership — parent/child relationships
Step 15

Make Corrections

Use the following actions to create training data. Each correction is saved immediately to the database:

ActionWhen to UseWhat Gets Saved
✅ Accept The detection is correct — right type, right box correction_type = "accept"
🏷️ Relabel Wrong element type — select the correct one from dropdown correction_type = "relabel" + new label
🗑️ Delete False positive — this region shouldn't be detected at all correction_type = "delete"
📐 Reshape Bounding box is inaccurate — drag corner/edge handles to fix correction_type = "reshape" + new bbox
↕️ Move Box is in the wrong position — drag to reposition correction_type = "reshape" + new bbox
➕ Add Missed region — draw a new bounding box and assign a type New detection + correction_type = "add"
🔗 Merge Multiple boxes should be one — select and combine into polygon Merged detection with polygon geometry
Step 16

Use Batch Operations for Speed

For efficiency on large pages with many detections:

  • Shift+Click — add boxes to multi-selection
  • Lasso Tool — drag to select a region of boxes
  • Then use batch Accept, Relabel, or Delete on all selected boxes at once
💡
Start by scanning the full page and deleting obvious false positives, then relabel misclassified regions, and finally accept everything that looks correct. This is the fastest workflow.

Step 18

Repeat Across Pages & Documents

Navigate to the next page and repeat the correction process. Process as many pages and documents as possible. The more corrections you make, the better your model will perform.

📊
Minimum requirement: You need at least 10 corrections (ml_min_training_examples) to train the first model. For good results, aim for 50+ corrections across multiple pages with good coverage of all 9 element types.
5

Train the ML Model Turn your annotations into a working classifier

Step 19

Option A: Train from the GUI

While still in the Annotation Tab, click the "Train Model" button. This triggers the full training pipeline in one click:

  1. Rebuilds the training_examples table from your corrections
  2. Applies stratified splitting: 70% train / 20% val / 10% test
  3. Exports to data/training_data.jsonl
  4. Trains the GradientBoostingClassifier (200 trees, balanced class weights)
  5. Optionally calibrates probabilities (isotonic calibration)
  6. Displays accuracy, F1, and per-class metrics in a popup
  7. Saves the model to data/models/element_classifier.joblib
  8. Records the training run in the database for experiment tracking
Step 20

Option B: Train from the Command Line

For more control, use the CLI training script:

# Basic training
PS> python scripts/train_model.py

# With ensemble (GBM + HistGBM + optional LightGBM/XGBoost)
PS> python scripts/train_model.py --ensemble

# Export to COCO JSON format (for external tools like Detectron2)
PS> python scripts/train_model.py --format coco

# Export to Pascal VOC XML format
PS> python scripts/train_model.py --format voc

# Force training even if F1 regresses from previous run
PS> python scripts/train_model.py --force

# Skip probability calibration
PS> python scripts/train_model.py --no-calibration

# Custom database path
PS> python scripts/train_model.py --db path/to/corrections.db
ℹ️
The CLI script includes an auto-rollback safety net: if the new model's F1 score is worse than the previous training run, it automatically restores the pre-training database snapshot (unless --force is used).
Step 21

Understand the Training Data

Here's what happens under the hood when training is triggered:

Training Data Format (JSONL)

Each line in data/training_data.jsonl is a single example:

Step 22

Verify Training Results

After training completes, check the output metrics:

  • Overall Accuracy — Percentage of correct predictions on the validation set
  • Per-class F1 Score — Harmonic mean of precision & recall for each element type
  • Confusion Matrix — (CLI only) Shows which types get confused with each other
💡
Low F1 on a specific class usually means you need more examples of that type. Go back to the Annotation Tab and add/correct more instances of the underperforming element type.

Step 23

Process New Documents with the Trained Model

Run the pipeline again on new (or the same) documents. The trained model now participates in the ML Feedback Loop, which runs 4 automatic passes after the core pipeline:

  1. Prior Corrections — IoU-based spatial matching carries forward previous corrections to matching regions
  2. ML Relabelling — If the classifier disagrees with the pipeline label at ≥ 80% confidence (ml_relabel_confidence), the label is overridden
  3. Confidence Scoring — Model confidence is attached to each detection
  4. Drift Detection — Statistical check against training feature distributions
Step 24

Monitor Model Health (MLOps Tab)

Switch to the MLOps Tab (Tab 4) via Ctrl+4 for ongoing model management:

SectionPurpose
Drift MonitorChecks if incoming data has shifted from what the model was trained on. Flags features exceeding the drift threshold (default: 0.3)
Retrain ControlShows accumulated correction count since last training. Alerts when threshold (default: 50 corrections) is reached. One-click "Retrain Now" button
Feature CacheView cache hit rates and invalidate stale feature vectors
Experiment HistoryFull table of all training runs with accuracy, F1, hyperparams. Export to CSV for external analysis
Step 25

Retrain Periodically

Retrain the model whenever:

  • The MLOps tab shows the correction count has exceeded the threshold
  • Drift detection alerts indicate feature distribution shifts
  • You've processed a new batch of documents with different characteristics
  • Per-class F1 shows clear weaknesses you've addressed with targeted annotations

Advanced Training Options Going beyond the base GBM classifier

Step 26

Enable Ensemble Mode

For improved accuracy with enough data, train a soft-voting ensemble:

PS> python scripts/train_model.py --ensemble

This combines multiple classifiers:

ModelRequired Package
GradientBoostingClassifierscikit-learn (included)
HistGradientBoostingClassifierscikit-learn (included)
LGBMClassifierpip install lightgbm
XGBClassifierpip install xgboost

Toggle in the GUI via the ML advanced config section: ml_ensemble_enabled = True.

Step 27

Enable Vision Features (Optional)

Add CNN image features to the feature vector for richer representations:

  • Set ml_vision_enabled = True in the ML config section
  • Choose backbone: ml_vision_backbone = "resnet18" or "efficientnet_b0"
  • Adds 512 dimensions from image crops of each detection region
Step 28

Enable Text Embeddings (Optional)

Add semantic text embeddings from a sentence transformer:

  • Set ml_embeddings_enabled = True
  • Default model: all-MiniLM-L6-v2
  • Adds 384 dimensions encoding the semantic meaning of extracted text
Step 29

Fine-tune LayoutLMv3 (Advanced)

For state-of-the-art layout detection using a transformer model:

# Install layout extras
PS> pip install -e ".[layout]"

# Prepare training data structure:
# data/layout_training/page_NNN/image.png + tokens.json

# Fine-tune
PS> python scripts/finetune_layout.py --data-dir data/layout_training --output-dir data/layout_model --epochs 10

Token format (tokens.json):

[
  {"text": "GENERAL NOTES", "x0": 50, "y0": 100, "x1": 250, "y1": 130, "label": "header"},
  {"text": "1. ALL WORK...", "x0": 50, "y0": 140, "x1": 450, "y1": 160, "label": "notes_block"},
  ...
]

Then set ml_layout_enabled = True and ml_layout_model_path = "data/layout_model".

Step 30

Train the Document GNN (Experimental)

For cross-page reasoning (e.g., consistent abbreviation tables across sheets):

# Install GNN extras
PS> pip install -e ".[gnn]"

# Train on a completed run
PS> python scripts/train_gnn.py --run-dir runs/run_20260221_103813_IFC_Operations_Facil --epochs 200
🧪
The GNN module is experimental. The page-result loader is currently a stub — it uses a synthetic demo graph. Full integration with real detections is still in development.
Step 31

Export for External Tools

Export your annotated data in standard formats for use with other ML frameworks:

# COCO JSON (Detectron2, MMDetection, etc.)
PS> python scripts/train_model.py --format coco

# Pascal VOC XML (YOLO converters, TensorFlow, etc.)
PS> python scripts/train_model.py --format voc

COCO export produces a single JSON with images, annotations (bbox + category_id), and categories. VOC produces one XML file per (document, page) pair.

Pre-Training Checklist

  • Virtual environment activated
  • At least one PDF processed through the pipeline
  • At least 10 corrections made in the Annotation Tab (50+ recommended)
  • Good coverage across multiple element types
  • Obvious false positives deleted
  • Misclassified detections relabelled
  • Correct detections accepted
  • Missing regions added with new bounding boxes
